## 📚 学习内容

### 1. Claude Code 核心概念

**关键概念**：
- **定位**：端到端的 AI 智能体，而非代码补全工具
- **能力**：独立完成开发任务、执行 Git 操作、运行测试、提交代码
- **上下文**：200K token 上下文窗口，可一次性加载整个代码库
- **界面**：终端原生设计，无需额外 IDE

**重要发现**：
- Claude Code 与 Cursor/GitHub Copilot 的本质区别：它是能独立完成任务的"初级工程师"级别智能体
- 核心优势在于完整闭环自动化：从代码编写到测试、提交的全流程
- 默认索引整个仓库，对大型项目和复杂任务处理能力更强

**代码示例**：
```bash
# 启动交互式会话
claude

# 直接执行命令
claude "创建一个用户登录的 REST API"

# Headless 模式快速输出
claude -p "分析性能问题" < script.py
```

### 2. Claude Code 最新功能（2024-2025）

**关键更新**：

#### Claude Opus 4.5 模型（2025年11月）
- SWE-bench Verified 得分 80.9%（业界领先）
- 支持 effort 参数控制推理深度
- 定价优化：$5/$25 每百万令牌

#### Agent Skills 功能（10月重大发布）
- **核心创新**：由 Claude 根据任务上下文自动激活
- **存储位置**：
  - 个人技能：`~/.claude/skills/`
  - 项目技能：`.claude/skills/`
- **市场规模**：186+ 个插件支持

#### Plugins 插件系统
- 包含：Slash Commands + Agents + Skills + MCP + Hooks
- 243+ 个可用插件
- 支持版本管理和程序化控制

#### Subagents 子智能体系统
- 专门化的 AI 助手，并行处理任务
- 独立上下文窗口
- 可限制工具访问权限

#### Hooks 钩子系统
- 自动化和定制工作流
- 可用事件：PreToolUse、PostToolUse、PermissionRequest 等
- 安全考虑：以用户凭证执行，需谨慎配置

### 3. 高级使用技巧

**自定义 Slash Commands**：
```markdown
# 文件：.claude/commands/analyze-performance.md
---
description: 分析代码性能问题并提供优化建议
---

请分析下列代码的性能问题：

$ARGUMENTS

提供以下分析：
1. 时间复杂度评估
2. 空间复杂度分析
3. 具体的优化建议
4. 重构代码示例
```

**Hooks 配置示例**：
```json
{
  "hooks": {
    "post-tool-use": {
      "matcher": "bash",
      "hooks": [
        {
          "type": "command",
          "command": "echo '✓ Changes staged' >> /tmp/claude.log"
        }
      ]
    }
  }
}
```

**MCP 服务器管理**：
```bash
# 查看已安装的 MCP 服务器
claude mcp list

# 安装新服务器
claude mcp add github --scope user -- npx -y @modelcontextprotocol/server-github
```

**CLAUDE.md 最佳实践**：
- 包含项目概述、技术栈、目录结构
- 记录常用命令和工作流程
- 定义编码规范和命名约定
- 列出要做和避免的事项

### 4. Claude Code vs 其他工具对比

| 特性 | Claude Code | Cursor | GitHub Copilot |
|------|-------------|--------|-----------------|
| 定位 | 端到端智能体 | AI 原生 IDE | 代码补全 |
| 界面 | 终端命令行 | 完整 IDE | IDE 扩展 |
| 上下文 | 200K tokens | Chunk-RAG | 当前文件 |
| 自动化 | 完整闭环 | 需手动触发 | 不支持 |
| 优势 | 大型项目、复杂任务 | 交互体验、成熟稳定 | 多 IDE 支持 |

## 💡 灵感与思考

- **核心认知**：Claude Code 不是"更强的 Copilot"，而是完全不同的开发范式——从"辅助编程"到"代理编程"
- **应用场景**：适合处理复杂的、多步骤的开发任务，特别是大型代码库的重构和优化
- **扩展性**：通过 Skills、Plugins、MCP、Hooks 的组合，可以打造高度定制化的开发环境
- **未来方向**：AI Agent 在软件开发中的角色会从"助手"演变为"协作者"
